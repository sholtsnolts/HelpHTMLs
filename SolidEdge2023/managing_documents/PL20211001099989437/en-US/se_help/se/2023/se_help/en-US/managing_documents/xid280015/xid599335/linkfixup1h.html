Fix links in Solid Edge files

Use this procedure to analyze your unmanaged Solid Edge files, and then fix broken links in the files before you import them into the managed environment.

Warning:

Failure to repair broken links before proceeding with the data preparation and import process can result in an improperly populated database.

  1. Choose Start→Siemens Solid Edge 2023→Choose PDM Integration.

  2. On the ribbon, click Utility.

  3. Select the document management application for which you are preparing your data:

    • Teamcenter

    • Unmanaged

  4. (Optional) To change the default location for the log files generated as output of the analysis, click Set Working Folder.

    Note:

    By default, the utility saves log files in \Users\<username>\AppData\Roaming\Siemens\Solid Edge\Version 223\Log Files\Data Preparation.

    To view the contents of the working folder click, Open Working Folder.

  5. In the Generate Import List tab, click Select Files/Folders.

  6. Select whether the files to be analyzed are located in a file or in a system folder.

    Use this option

    To do this

    Add Files

    Identify individual files for analysis or define the location of either the ordered or unordered CSV file containing the list of files for analysis.

    Add Folders

    Identify the location of the system folders containing the files for analysis.

  7. Browse for the files or folders specified in the previous step.

    Tip:

    • When you choose to Add Folders, you can select the Include Subfolders check box so that any subfolders are included for you.

    • You can click Types to filter your selection by file types (.dft, .asm, .par, .psm, .pwd or All documents including non-Solid Edge documents).

    • You can click Remove or Remove All to modify the list of files or folders.

  8. To start the analysis, click Generate Import List.

    The length of time required for processing depends on the number of files analyzed. When broken links are found, a message is displayed. You have the option to:

    • View Broken Links

      Provides details about the broken links along with alternate paths for the link.

    • Continue

      Proceeds with the analysis.

    • Cancel

      Stops the analysis.

  9. Click View Broken Links.

    You are provided details about the broken link and alternate paths.

  10. Choose the View Properties tab.

    The path to the broken links file, BrokenLinkLog_<timestamp>.xml, is provided for you.

  11. Verify the path to your Unordered_<timestamp>.CSV file is correct.

    Both the BrokenLinkLog_<timestamp>.xml and the Unordered_<timestamp>.CSV were generated by the file analysis.

  12. Click Analyze.

  13. Your analysis information can be spread across multiple datasheets in PropertyList_<timestamp>.xlsm. Select the datasheet that contains the link replacement information.

  14. For each broken link, provide both the broken link path and broken link file name as well as the replacement path and replacement file name.

  15. Save the spreadsheet and click Fix Broken Links.

    After the status indicates Finished processing, you can continue modifying properties or use Design Manager to verify that the broken links are fixed.

  16. In the Data Preparation Utility, select the Modify Properties tab and click Modify Properties.
