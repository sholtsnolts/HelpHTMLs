Managing translated files with Teamenter

During the life cycle of a product, there is often a need to save a document in a translated (non-Solid Edge) format so that vendors can use it. For example, you may need to send a part out to a manufacturer who requires the part document in STEP (.stp) format.

In the Teamcenter-managed environment, you can use the Save as Translated command to save Solid Edge documents to the following formats:

With Save as Translated, you can also determine whether the translated files are saved to Teamcenter (managed) or to disk (unmanaged) by setting the option, Save Translation to Teamcenter.

When you choose to save translated files to Teamcenter, you define how you want the translated files saved by using options on the Teamcenter - Save As Translated dialog box. Translations are saved in the same Item Revision as the Solid Edge file, however, you must define:

  • File type (example: Parasolid, STEP)

  • Relation (example: IMAN_manifestation, IMAN_specification)

    Associates the Dataset type to the Item Revision.

  • Dataset type (example: Image, DXF)

    Specifies the container in Teamcenter that holds the file.

  • Named Reference

    Identifies the file in the Dataset.

The file name is automatically generated for you in the format ItemID_originalExtension.ext. For example, if 000237/A is exported to a STEP format, the assigned file name is 000237_asm.stp. You can customize the file name by defining preference values in the Teamcenter preference SEEC_Save_Translation.

Defining the behavior for managed translated files

The Teamcenter preference, SEEC_Save_Translation determines:

  • Whether translations can be saved directly to disk or to Teamcenter.

  • Enabling of the automatic translation of Draft files (DXF, DWG, and PDF only) and flat patterns in Part or Sheet Metal files when a flat pattern is present (DXF only).

  • The translation options available to you on the Teamcenter - Save As Translated dialog box.

  • Available search criteria for extraction of translated files.

For a full description of the preference, see the Teamcenter Preferences section of the Teamcenter Integration for Solid Edge (SEEC) Guide for Users and Administrators.

Deferring translation to Teamcenter Dispatcher

You can select multiple files for translation and defer the translation process from Solid Edge to a managed request in Teamcenter Dispatcher. Define the SEEC_Dispatcher_Translation preference to generate a Dispatcher request and process the translation outside of Solid Edge. When the preference is disabled, the object name of the object being translated is shown on the Teamcenter - Save As Translated dialog box. With the preference enabled and:

  • No document is selected - translation is performed on the active document You have the option to translate the document immediately in Solid Edge or defer the translation to Dispatcher.

  • A single document is selected in an assembly structure - translation is defered to Dispatcher.

  • Multiple documents are selected in an assembly structure - translation is defered to Dispatcher.

For a full description of the preference, see the Teamcenter Preferences section of the Teamcenter Integration for Solid Edge (SEEC) Guide for Users and Administrators.

Extracting translated files from Teamcenter

Translated files stored in Teamcenter can be collected for sharing using an extraction process that can be performed while your work in Solid Edge, or while no document is open. When you are working in Solid Edge, use options in the Extract Translated Files pane. When you want to extract the translated files without opening a document, choose File menu→Manage→Extract Translated Files. Extracting the translated files does not create a package or zip of the files. It only places the files in the location you specify.
