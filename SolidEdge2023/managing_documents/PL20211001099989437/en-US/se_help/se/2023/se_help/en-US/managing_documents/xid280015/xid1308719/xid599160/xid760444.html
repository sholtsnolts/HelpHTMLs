Reimporting documents using Add to Teamcenter Interactive

Documents that are candidates for reimport to Teamcenter are generally those that have been loaded into Teamcenter, then later modified by a vendor, and returned for loading into Teamcenter with modifications. The Add to Teamcenter Interactive program compares the item number, revision, and dataset values of a document with those existing in the Teamcenter database. As a result, you can import files over files that already exist in Teamcenter.

Add to Teamcenter Interactive adds the import time to the original file through the use of a custom property, TC_ImportTime. On subsequent imports it checks to see if the modified date is after the import date. If the document being imported has not been modified since the last import, it is recognized as already being in the database and you are given the option to use the existing file. If the unmanaged file is modified after its initial import into the database, on subsequent imports you are given the options:

  • Existing – Use Existing

    This action is automatically set for you if the document was previously imported and it has not changed since it was last imported.

  • Existing – Overwrite

    This action is automatically set for you if the document was previously imported but was modified since it was last imported. Choosing this command overwrites existing Teamcenter data.

    Note:

    During an overwrite condition, BOM precision is not changed.

  • Existing – Revise

    The document was previously imported and a new revision is needed.

  • Existing – Save As

    The document was previously imported and you wish to save a copy to a new item number, revision, and item name.

  • New

    The document was not previously imported.

Also, in the case where multiple draft files or multiple Microsoft Word, Excel or image datasets are saved to the same Item Revision, the program adds a custom property, PDMFilename, to the unmanaged file and sets its value equal to the managed dataset’s filename. It can then use that custom property to determine if the file being reimported is new or exists in Teamcenter by another name.

Note:

The custom property does not exist in managed files.
