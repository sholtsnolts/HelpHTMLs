Create and replace document revisions

If you are a Solid Edge data management user, you can:

  • Create a revision of the active document using the File menu→Save As→Save As New Revision command .

  • Create and replace revisions of documents selected in PathFinder using the Data Management tab→Revisions command .

If you move revisions manually, links to those revisions break and the system informs you that the revisions are missing. You can fix broken links using the Broken Links command in Design Manager.

Create a document revision using the Save As New Revision command

Use this procedure if you just want to create a new revision of the part or assembly document that is open in Solid Edge.

  1. Open the document of interest.

  2. Choose File menu→Save As→Save As New Revision .

    Depending on the selections you made in the Solid Edge Options dialog box→Manage tab→Document Naming Rule page, you have the following options:

    • If the check box Automatically name files via Document Number and Revision is not selected, the system displays a dialog box showing an incremented revision value.

    • If the check box Automatically name files via Document Number and Revision is selected, the system displays a dialog box, showing an incremented revision value, where you can modify the document location, revision number, and title, as explained in Creating documents with unique properties.

  3. Click Save.

    The system creates the document revision, closes the current document revision, and opens the newer document revision in Solid Edge.

Create a document revision using the Revisions command

Use this procedure if you want to create a new part document revision and replace the selected document with the newly created revision.

  1. In PathFinder, select the part document for which you want to create a new revision.

  2. On the ribbon, on the Data Management tab, click Revisions .

  3. Click New.

    Depending on the selections you made in the Solid Edge Options dialog box→Manage tab→Document Naming Rule page, you have the following options:

    • If the check box Automatically name files via Document Number and Revision is not selected, the system displays a dialog box showing an incremented revision value.

    • If the check box Automatically name files via Document Number and Revision is selected, the system displays a dialog box, showing an incremented revision value, where you can modify the document location, revision number, and title, as explained in Creating documents with unique properties.

  4. Click Save.

    The system creates the document revision and replaces the selected document with the new revision.

Replace a document with a previous revision

Use this procedure if you want to replace a new part document in an assembly with a previous revision.

  1. In PathFinder, select the part document you want to replace.

  2. On the ribbon, on the Data Management tab, click Revisions .

  3. From the list at the bottom of the dialog box, select a new revision to replace the one in PathFinder.

    Note:

    Revisions created in previous Solid Edge releases are not available in this list.

  4. (Optional) To open a previous revision in a new tab, right-click the revision and click Open.

  5. (Optional) To see more information related to the selected revision, right-click the revision and use the Preview, Properties, or Where Used options.

  6. Right-click the selected revision and click Replace.

    Solid Edge does one of the following:

    • If it finds just one instance of the document in the assembly, it replaces it in PathFinder with the revision you chose.

    • If it finds two or more instances of the document in the assembly, it prompts you to indicate if you want to replace just the selected document or all instances of the document.
