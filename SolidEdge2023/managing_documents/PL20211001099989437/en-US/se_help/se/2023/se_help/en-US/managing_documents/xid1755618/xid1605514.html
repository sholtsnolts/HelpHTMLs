Obsolete older revisions

For better design and implementation control, you can now specify if you want to have only one released version of a part in the system. This functionality automatically makes sibling and parent revisions obsolete when you release the latest revision from the Solid Edge data management vault. If you set life cycle file locations, Solid Edge moves the obsolete files to their designated locations, while keeping the folder structure intact. For more information on setting file locations, see Define file locations for release management.

When you choose to obsolete older revisions, Solid Edge also updates links on the where-used files to avoid broken links, generates a CSV report about the file changes, and stores that report in C:\Users\<username>\AppData\Roaming\Siemens\Solid Edge\Version 223\Log Files. If the format of your CSV file is not correct, change the region setting in the Control Panel. For more information, see https://www.itsupportguides.com/knowledge-base/office-2013/excel-20132016-how-to-change-csv-delimiter-character/.

You set the option Automatically obsolete previous revisions while releasing a file in the Manage tab, Solid Edge Options dialog box.
