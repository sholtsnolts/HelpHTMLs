Working Offline in the managed environment

Under certain circumstances, your workflow may require that you work without access to a managed environment. For example, you may need to take a laptop computer out of the office to a customer site, to the shop floor, or you may need to work with a collaboration partner. Working offline enables you to work with Teamcenter-managed documents when you are not connected to the server.

When you work offline, you cannot perform actions that require transactions with the server. For example, you cannot check out documents. Instead, you work with documents that are in your local user cache.

Preparing to work offline

It is important to prepare your data before working offline.  Before going offline you should:

  • Download any documents that you will need for the offline session.

    Use the Download command to download any documents to the local user cache that you will need during your offline session.

  • Check out any documents to which you will need write access during the offline session.

    Check out any documents that you are going to use during the offline session to gain write access to the documents. If you do not have write access to the documents you will not be able to modify them.  You can use the Check Out command to check out the documents you need.

  • Synchronize any documents that you will need that are already in your cache.

    Teamcenter-managed documents in your local user cache can be synchronized with the contents of the server by clicking Synchronize in the Cache Assistant ribbon.

Going offline

In the Teamcenter-managed environment, open the Cache Assistant, choose Work Offline, and close the Cache Assistant dialog box.

Note:

The Teamcenter preference SEEC_Work_Offline must be enabled for you to have the option to work offline.

You also have the option to set your working mode to Always work offline when Solid Edge starts by selecting the option on the General tab of the Solid Edge Options→Manage page.

Working offline with Teamcenter-managed documents

When you are working without access to Teamcenter, commands that interact with Teamcenter are disabled. However, you can open and work with documents you have checked out, or create new documents you want to check in to Teamcenter later. The following indicators let you know you are working offline:

  • The application title bar displays Teamcenter - Working Offline.

  • The Cache Assistant title bar displays Cache Assistant - Working Offline.

  • User Session Information indicates [Working Offline].

  • The Read-Only Assistant displays a message that you are working offline.

  • The Open File dialog and Quick Search interacts with the contents of your local user cache.

As you create and save new documents while you work offline, the documents are saved to your local cache. A temporary filename is assigned to each document indicating the document was created while you were offline from Teamcenter. For example, offline-102831.par. When you upload or check the document in to Teamcenter, property data is assigned to the document.

You can choose to display the File Properties dialog box the first time a document is saved in offline mode. Enable the option on the General tab of the Solid Edge Options→Manage page.

Returning online

In a Teamcenter-managed environment, you can choose to have your cache synchronized with Teamcenter and have documents checked in automatically when you return online. Set the Check In All command to check in all checked out documents when you reconnect with Teamcenter, or set the Check In New command to check in any new documents you created while offline.

To return to online mode, click Work Online and restore your connection with Teamcenter. If you had previously logged into Teamcenter, you are not prompted to log in again.
