Publish concepts to Teamcenter

Documents created or modified in Concept Workspace are added to Teamcenter using the Concept Assistant→Publish command.

Note:

Publishing Simulation (ssd) files created or modified in Concept Workspace is not supported.

  1. In Concept Workspace, open the document (.asm or .par) you want to publish to Teamcenter.

    Any document with a status of New or Modified, along with the associated drafts, is published. Documents with a status of Up to Date are not published.

  2. Specify attributes that will be passed to the upload process. For example, Action and Item Type.

  3. To determine if there are any conflicts based on the defined attributes, click Validate.

    For information on resolving any issues, see Resolving problems identified in the Issues column (Concept Assistant).

    Note:

    A tip indicating a newer version is available implies that a component has changed in Teamcenter since you started your concept design. Run the Compare command for a 3D comparison between your concept and the version in Teamcenter. You can then choose from the actions: Upload, Save Item As, Revise, New Item, or Revert.

  4. To check for a newer version of a non-modified object, in the Solid Edge application window, click Teamcenter tab→Update Status Info.

    If a newer version exists, use the Compare command to determine which version of the document you want for your design. If you prefer the Teamcenter version, no action is required. If you want the non-modified version from cache, in-place active the component and save it. The status of the object is changed to Modified and you can choose one of the following actions: Upload, Save Item As, Revise, New Item, or Revert.

  5. Click Publish.

    The Solid Edge session is closed and the Concept Assistant Publish dialog box is displayed. Property data is shown based on the currently displayed columns.

    Note:

    Publishing conceptual designs to Teamcenter is similar to the processes for Add to Teamcenter-Interactive. For information on Add to Teamcenter-Interactive, see Getting started with Add to Teamcenter Interactive.

  6. Determine the parameters for this session by clicking Session Settings , and then selecting the options from the Session Settings dialog box.

  7. Evaluate the readiness of the documents you selected for loading into Teamcenter.

    The Issues column displays a cell for each document in either green or red to indicate the document’s readiness to be input to Teamcenter. Red cells indicate problems that should be considered before continuing with importing the documents to Teamcenter.

  8. Define the required properties for new documents by clicking Assign All or right-clicking and selecting Assign.

  9. Click the Run Teamcenter Validations command to validate the state of your documents.

  10. Consult the Error Assistant to review and correct any reported problems.

  11. Click Perform Actions .

    A log file is generated that contains details about the transaction. The Progress Log summarizes whether the documents passed or failed import to Teamcenter.

  12. Successful import of your documents into Teamcenter completes the session.
