Resolving data issues

The longer you work with your CAD files in an unmanaged environment, the higher the likelihood that your data is inconsistent. Examples of inconsistencies include:

  • Duplicate file names—Someone copied an assembly to a new folder, maybe as a starting point, and copied some parts too. Now the same part exists in two places.

  • Broken links—You moved a file and now a parent assembly (or a Part Copy Feature) can no longer find it.

To resolve these types of inconsistencies, use the following commands, located on the Tools tab:

  • Duplicate Names

  • Broken Links

  • Redefine Links
