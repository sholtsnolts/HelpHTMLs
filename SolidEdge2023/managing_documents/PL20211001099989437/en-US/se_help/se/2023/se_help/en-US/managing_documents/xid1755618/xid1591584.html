Releasing documents (Solid Edge data management)

Using the Solid Edge data management workflow functionality, you can automate business processes in which documents are passed from one user to another. Workflows can involve multiple users who review or approve, and often result in a change of status. Out of the box, Solid Edge data management offers two workflow types.

Because workflows automatically generate a list of references for an assembly and set the status, you eliminate the need to manually release the child documents of an assembly before releasing the assembly. The system also automatically sets the linked draft document status along with that of the selected document, so you no longer have to perform a where-used search for linked draft documents.

When you release a document, Solid Edge can automatically make sibling and parent revisions obsolete. For more information, see Obsolete older revisions.

You can also release documents in Design Manager. For more information, see Releasing documents (Design Manager).
