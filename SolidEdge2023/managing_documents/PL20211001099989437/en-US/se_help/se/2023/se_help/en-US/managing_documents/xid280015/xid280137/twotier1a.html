Solid Edge Embedded Client two-tier configuration

Solid Edge Embedded Client implements Teamcenter's Service-Oriented Architecture (SOA) and File Management System (FMS). This implementation can be used with Teamcenter's four–tier deployment, where SOA is a component of this solution, or it can be configured to be used with a Teamcenter two–tier deployment.

In a four–tier deployment, only the client application executes on the user's workstation; the Enterprise tier, database, volume, and other resource tier components are executed on the server providing improved performance for customers working in a high-latency WAN environment. By default, Solid Edge Embedded Client is a four–tier configuration.

Note:

The cache is on the selected Teamcenter Database URL. Therefore, a four–tier connection does not use the same physical cache as a two–tier connection for the same Teamcenter database and user ID.

If a Solid Edge Embedded Client two–tier deployment is required, Solid Edge Embedded Client should be configured after setup is complete. Use the Define Database Definition application to create a two–tier database definition.

In preparation for configuring a two–tier environment:

  • Install a Teamcenter two–tier rich client. The installation delivers the required Teamcenter application support for an SEEC two–tier deployment.

  • Apply all the Teamcenter patches to meet the minimum Teamcenter application requirements.

  • Verify your Rich Client installation.

To learn more, see the Help topic, Configure Solid Edge Embedded Client for two-tier deployment.
