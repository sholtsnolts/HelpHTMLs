Configuring hosted Active Workspace

Before you can use hosted Active Workspace, your browser must meet the minimum requirements and the associated Teamcenter preferences must be defined.

Browser requirements

Hosted Active Workspace requires at least Internet Explorer 11. However, if Chromium-based Microsoft Edge is discovered, it is used instead.

To see a complete list of software compatibility for Solid Edge 2023 with the Teamcenter integration, see the Software Field Bulletin for Solid Edge and the Teamcenter Integration. A WebKey is required to access the site.

Note:

The Solid Edge file type .seectcxml is downloaded by your browser. Opening this file accesses the document you requested. Refer to your browser's documentation to enable automatic opening of this file type.

Teamcenter preferences

The following Teamcenter preferences must be configured on the Teamcenter server to use hosted Active Workspace:

AWC_SEEC_OpenSupportedTypes

Enables the display of the Open in Solid Edge command and specifies the object types that can open in Solid Edge. Currently, Item Revision and Dataset types are supported, along with custom Item and Item Revision item types. Limit the preference values to use only objects you would use in Solid Edge. The preference is read once per session. You must close and reopen the Active Workspace client after editing preference values to see the effects of a change.

ActiveWorkspaceHosting.SEEC.URL

Defines the URL used by Solid Edge when hosting the Active Workspace client.

For more information about Teamcenter preferences, see the Teamcenter Integration for Solid Edge (SEEC) Guide for Users and Administrators.

Options for opening documents

When you open Teamcenter-managed documents from the hosted Active Workspace client, the options, Always show the Open dialog box and Always ask before inserting the model into the active assembly determine the behavior you experience. You can set these options on the Open tab, Manage page (Solid Edge Options) dialog box.
