Workflow types

Out of the box, Solid Edge data management offers two types of workflows.

One Step Release workflow

The One Step Release workflow routes the active document to a reviewer. After the reviewer approves or rejects the task, the workflow assigns the status Released or Available to the document.

Note:

This workflow type is available only if you open the document you want to release from a shared location (UNC path) you specified when you defined the vault.

Only documents with In-Review status can be approved or rejected. In addition, only documents with the following status type can participate in workflows:

  • Available

  • In-Work (only if worked on by the same user who initiated the workflow)

  • In-Review

Using the One Step Release workflow you can:

  • Approve or reject changes.

  • Change the status of files.

  • Update drafts.

  • Create PDF and JT files, as well as other neutral file formats.

  • Create reports, such as lists of parts, for released assembly documents.

  • Automatically notify others via email about workflow actions or status changes. To be notified, you must add their email addresses to the Workflow Reviewers list, as explained in Define workflow options.

Quick Release workflow

The Quick Release workflow assigns the status Released to the active Solid Edge document in a single step. To learn how to send documents through the Quick Release workflow, see Release documents (Solid Edge data management) or Release documents (Design Manager).
