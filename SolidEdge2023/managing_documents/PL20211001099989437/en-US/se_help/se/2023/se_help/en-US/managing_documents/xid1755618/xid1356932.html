Create documents with unique properties

To avoid creating duplicates when you create new files, you can use the Automatically name files via Document Number and Revision option to create documents with unique file names, document numbers, and revision numbers. This option ensures file uniqueness for assemblies, parts, and drafts during any new part creation process. It is available to Solid Edge data management users from the Solid Edge Options dialog box→Manage tab→Document Naming Rule page.

Use the following workflow to create documents with unique properties:

  1. Set up Solid Edge data management as explained in Getting started with Solid Edge data management.

  2. On the Solid Edge Options dialog box→Manage tab→Document Naming Rule page, select the Automatically name files via Document Number and Revision option.

    You can specify a prefix in addition to the number specified in the NumberGenerator.dat file. All document numbers are incremented from the number in this file, therefore the file must not be empty. The incremented number must only include numeric values. We recommend that you use the Document Naming Rule options on the Manage tab, Solid Edge Options dialog box to define the increment value.

  3. Perform any Solid Edge operation that creates new files, such as Save As, Replace Part with Copy, or Mirror Component.

    Solid Edge displays a dialog box where you can assign common properties and more:

    • To specify where you want to save the new document, either enter the location in the Location box, or double-click the box to display Windows Explorer and navigate to the location of interest.

    • To automatically generate and assign a document number, click Assign All .

    • To verify that your entries are correct, click Validate .

    • To save the new document, click Save .

    For more information, see Assigning common properties.
