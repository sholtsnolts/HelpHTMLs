Change the default Teamcenter project

The Teamcenter preference SEEC_ProjectID_Default determines whether the default is the last Project ID specified in CPD, or the Project ID defined in the Session page of the User Settings dialog box in Solid Edge, Hosted Active Workspace, or Teamcenter RAC.

Note:

For a full description of the preference, see the Teamcenter Preferences section of the Teamcenter Integration for Solid Edge (SEEC) Guide for Users and Administrators.

To set the default Project using the User Settings dialog box:

In Cache Assistant

  1. From the File menu, click Manage→Cache Assistant.

  2. Click the ID Link shown in bold.

  3. In the User Settings dialog box, select a project from the Project list.

    Note:

    You must have privileged access to at least one project in Teamcenter. If no projects are defined in Teamcenter, the Project list is empty.

  4. Click OK.

  5. Close the Cache Assistant dialog box.

In the Solid Edge startup page

  1. Start Solid Edge.

  2. Select Click here to sign in....

  3. Enter your User ID and Password, and then click Sign In.

  4. Click the ID Link shown in bold.

  5. On the Session page of the User Settings dialog box, select a project from the Project ID list.

    Note:

    You must have privileged access to at least one project in Teamcenter. If no projects are defined in Teamcenter, the Project list is empty.

  6. Click OK.

On the User Session pane

  1. With a document open in Solid Edge, click the User Session Info pane.

  2. Click Start User Settings dialog box.

  3. On the Session page of the User Settings dialog box, select a project from the Project ID list.

    Note:

    You must have privileged access to at least one project in Teamcenter. If no projects are defined in Teamcenter, the Project list is empty.

  4. Click OK.
