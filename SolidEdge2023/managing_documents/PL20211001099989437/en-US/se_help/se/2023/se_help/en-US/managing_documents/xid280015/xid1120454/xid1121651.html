Starting a Teamcenter workflow

In a collaborative work environment, sometimes items must pass from one participant to another in a manner that is controlled by rules or procedures. Teamcenter provides the capability to manage your product data processes to accommodate your business practices. The automation of the process is considered a workflow. Workflows typically apply a release status, send items to appropriate users, route approvals and notifications to stakeholders, and manage the change management processes.

You can select item revisions for the workflow process from Solid Edge using the New Workflow Process command.

You can view the Teamcenter release status of documents in the Release Status column of various dialog boxes.

To track the stage the Item Revision is in as it travels through the workflow, view the Process Stage column that is displayed in various dialog boxes such as the Open File dialog box (SEEC). In this column, the value of the Teamcenter property process_stage_list is displayed in a comma-separated list from current stage to previous stage. The last item in the list states the workflow name. If the document is not part of a workflow, or has completed a workflow, the property is empty.

Use the Teamcenter rich client or Teamcenter Active Workspace to view workflow tasks.

Note:

To interact with workflows, use Hosted Active Workspace, Active Workspace, or the rich client.
