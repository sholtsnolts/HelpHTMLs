Restart a session (workflow)

Add to Teamcenter Interactive maintains transaction integrity in the event the application experiences a problem that interrupts your session. For example:

  • Prior to selecting Perform Actions the application is interrupted, upon restarting the application you are prompted that actions have not been completed and you are given the opportunity to either open the last saved session and complete the actions, or discard the last saved session completely.

  • After selecting Perform Actions, while the actions are being performed, the transaction is interrupted. Upon restarting the application you are prompted that actions need to be completed before a new session can be started. You can click restart to complete the remaining actions, suspend to take no action, or discard to not complete the remaining transactions and start a new session.

Workflow for restarting a saved session

Use the following process to save and then restart the saved session in Add to Teamcenter Interactive.

  1. During an active session, click File→Save Session.

  2. Exit Add to Teamcenter Interactive.

  3. Restart the application.

  4. Respond to the prompt to either open and complete remaining actions, or discard remaining actions.

    You can click Cancel to take no action and dismiss the message box.
