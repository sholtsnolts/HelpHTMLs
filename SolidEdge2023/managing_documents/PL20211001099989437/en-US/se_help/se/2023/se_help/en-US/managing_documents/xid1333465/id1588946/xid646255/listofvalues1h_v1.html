Define the List of Values (LOV) value-description separator

The Value-Description Pair Display defines the separator character used between values and their descriptions. You can specify whether to display descriptions along with a particular list of values. It is recommended you use one of the following characters: dash (-), at symbol (@), pound symbol (#), or ampersand (&).

  1. Start the Teamcenter Rich Client.

  2. Choose Edit→Options and click List of Values.

    Note:

    The Teamcenter Preferences that impact Lists of Values are described in the Teamcenter Integration for Solid Edge (SEEC) Guide for Users and Administrators.

  3. In the Separator Between Value and Description field, type the character you want to use.

  4. To hide the description when value-description pairs are used for a list of values, choose the value in the Shown list and move it to the Hidden list.

  5. Click Apply or OK.

Note:

If the value and description are attached to two different properties of a single type, and the list of values description is shown, the value and description are displayed side-by-side in both lists. However, when the selection is made, the Description field displays only the description rather than the value and description combination.
