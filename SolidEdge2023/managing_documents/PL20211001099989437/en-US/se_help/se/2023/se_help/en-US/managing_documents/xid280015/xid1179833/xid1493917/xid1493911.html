Configuring standalone Active Workspace

Before you can use standalone Active Workspace, your browser must meet the minimum requirements and the associated Teamcenter preference must be defined.

Browser requirements

Standalone Active Workspace requires at least Internet Explorer 11, but is also supported with Chrome, Firefox, and Safari.

Note:

The Solid Edge file type .seectcxml is downloaded by your browser. Opening this file accesses the document you requested. Refer to your browser's documentation to enable automatic opening of this file type.

Configuring standalone Active Workspace

To use the standalone Active Workspace client, you must select Integration for Solid Edge during the Active Workspace installation through Teamcenter Environment Manager (TEM), on your Teamcenter server.

The Teamcenter preference used by standalone Active Workspace is:

AWC_SEEC_OpenSupportedTypes

Enables the display of the Open in Solid Edge command and specifies the object types that Active Workspace can open in Solid Edge. When the preference is not present, the Active Workspace does not display the Open in Solid Edge command. Item Revision and Dataset types are supported, along with custom Item and Item Revision item types. Limit the preference values to use only objects you would use in Solid Edge. The preference is read once per session. You must close and reopen the Active Workspace client after editing preference values to see the effects of a change.

For more information about Teamcenter preferences, see the Teamcenter Integration for Solid Edge (SEEC) Guide for Users and Administrators.
