New Workflow Process command

Initiates an automated process defined in Teamcenter, where documents are passed between participants of the workflow to accomplish a particular objective. The item revisions of the documents that are chosen for the workflow process are referred to as targets.

The command supports single and multiple document selection from PathFinder in the assembly environment. Direct and indirect documents are accepted into a selection set. The selection set is submitted to a single workflow.

You can track the stage the Item Revision is in as it travels through the workflow by viewing the Process Stage column that is displayed in various dialog boxes such as the Open File dialog box (SEEC). In this column, the value of the Teamcenter property process_stage_list is displayed in a comma-separated list from current stage to previous stage. The last item in the list states the workflow name. If the document is not part of a workflow, or has completed a workflow, the property is empty.

View the status of documents within Teamcenter workflows in the Release Status column of various dialog boxes.

Use the Teamcenter rich client or Teamcenter Active Workspace to view workflow tasks.
