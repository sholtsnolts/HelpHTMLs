Using Teamcenter in Solid Edge

The Teamcenter Integration for Solid Edge, Solid Edge Embedded Client (SEEC), provides connectivity between Solid Edge and Teamcenter. The integration is transparent, and you do not need to learn a separate user interface or additional commands to use it. Solid Edge commands and supporting tools that create or close documents interact with the Teamcenter data structure to manage those documents so that you do not have to.

Installing the Teamcenter Integration for Solid Edge (SEEC)

Installation of Teamcenter Integration for Solid Edge (SEEC) occurs when you install Solid Edge. Information regarding configuration is available in the Teamcenter Integration for Solid Edge (SEEC) Guide for Users and Administrators. This document, SEECAdminGuide.pdf, is available from Solid Edge Help.

Working with Teamcenter in Solid Edge

When you have a license for the Teamcenter Integration for Solid Edge, you are automatically started in the Teamcenter managed environment. The Solid Edge features that work with Teamcenter are enabled. For example, Teamcenter is indicated in the application title bar of the window, and when you browse for Solid Edge files you are presented with the data from Teamcenter.

You can disable or enable Teamcenter mode by clicking the File tab, and choosing Manage→Teamcenter.

Note:

The command is disabled if there are files open in Solid Edge.

If Teamcenter is not indicated in the title bar, you are working in an unmanaged Solid Edge environment.

Typical workflow for managing Solid Edge documents in Teamcenter

  • Create a new Solid Edge document and save the document.

  • Close and check in or upload the document to Teamcenter.

  • Open the document to check it out of Teamcenter.

Use options on the General tab in the Manage page of the Solid Edge Options dialog box to determine how Teamcenter-managed documents are closed or revised.
