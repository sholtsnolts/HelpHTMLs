Search for duplicate names

You can use the Duplicate Names command to search for all documents that have duplicate names and create a text file that lists the names and locations of duplicate documents. You can search for documents anywhere on your network, including a managed library.

For optimum performance, managed queries are limited to 20 documents per document library and 30 document libraries. You can modify these limits by editing the registry keys NUM_DOCS_PER_QUERY and NUM_DOCLIBS_PER_QUERY. When you open a managed document, these registry keys are generated automatically in HKEY_CURRENT_USER\Software\Siemens\Solid Edge\Version 223\DEBUG.

  1. Choose Tools tab→Search group→Duplicate Names .

  2. Depending on the product data management (PDM) integration client you are using, you have the following options:

    • If you are using Teamcenter Integration for Solid Edge (SEEC), the system displays a dialog box where you specify the search scope.

      1. In the Look in box, specify the drive and folder you want to search.

      2. From the list on the left, select the document you want to search and then click Add.

        In the list Search for duplicate document names in, red indicates non-indexed folders.

      3. Repeat the previous two steps until all locations of interest are added to the search list.

      4. (Optional) Do any of the following:

        • To remove one or all documents from the search list, click the Remove or Remove All button, respectively.

        • To include subfolders in the search, select the Include subfolders check box.

        • To select file types as search criteria:

          • Click Process Options

          • Select the specific file type and then click OK.

      5. Click Next.

    • If you are using Solid Edge data management and have defined a vault containing your Solid Edge files, the system automatically searches your specified vault locations for any Solid Edge files. The system does not display the dialog box where you specify the search scope.

    The system displays information about how many documents it processed and how many duplicate documents it found.

  3. (Optional) Do any of the following:

    • To view a log file of the results, click View.

      You can use the information from this log file to resolve the issue of duplicate names.

    • To cancel and close the dialog box, click Cancel.

    • To perform another search, click Back.

  4. To finish the operation, click Finish.

  5. In Windows Explorer, rename the duplicate files and then run the Duplicate Names command again to verify that you have resolved the issue.
