Configure the Geolus server to index Solid Edge files

Administrators of the Geolus server must configure Geolus for indexing Solid Edge documents. Attributes must be defined as shown.

Note:

The information below does not apply to users who manage content in Teamcenter.

  • Refer to Geolus documentation to configure the following mandatory file properties as generic attributes on your Geolus server. Property names are case-sensitive.

    • FilePath

    • ModifiedTime

    • DocName

  • (Optional) Configure the following properties as generic attributes on your Geolus server. The ShapeSearchMapping.txt file can be used to map the real name (as shown) to a different display name (that displays in the Geolus web client).

    • Title

    • Document Number

    • Revision

    • Project Name

    • Creation Date

    • Last Save Date

    • Last Author

    • Size

    • Status

    • StatusDate

    • StatusUserDisplay

    • Hardware

    • Material

    • Density

    • GeometricVersion

  • Initialize the Geolus server database with the attribute names you configured in the previous steps.

    • Edit the Geolus database table named T_ATTRIBUTE_NAMES.

    • Add the attribute names you configured previously.

  • Credentials that have both read and write access to the Geolus database must be specified for use by batch as well as online processes. This can be accomplished by running the Geolus Core Configuration utility on the Geolus server. By default, this username and password is: U_GEOLUS_BATCH. Enter these credentials under the Geolus Database Users page.
