Saving the surface association of PMI in part files

The surface information associated with model PMI annotations and dimensions in a part or sheet metal file is used by third-party applications such as NX Inspection, Femap, and Solid Edge CAM Pro. This surface association is stored as lite part data. Lite part data is an internal object of Solid Edge part, sheet metal, and assembly files. Lite part data is read by third-party tools using the lite part adapter XML, which is provided by Solid Edge with every release.

A registry entry controls whether PMI parent surface information or parent edge information is saved with the file and made available to non-native applications.

  • In the folder, Computer\HKEY_CURRENT_USER\Software\Siemens\Solid Edge\Version 223\DEBUG

  • Locate the registry switch:

    Name=SAVE_LITE_PMI_SURFACE_ASSOCIATION

    Type=REG_DWORD

    Value=1 (surface association) or 0 (edge association)

Note:

  • The default value of the registry switch upon installation of Solid Edge=1 (surface association).

  • Solid Edge users who open their own files in NX and Solid Edge CAM Pro can specify that PMI primarily reference parent edges by changing the registry value=0 (edge association).

Exporting internal components

You can export Solid Edge files containing internal components. This allows you to:

  • Open Solid Edge files in NX and Solid Edge CAM Pro.

  • Export Solid Edge files to JT using SePvAdp.exe.
