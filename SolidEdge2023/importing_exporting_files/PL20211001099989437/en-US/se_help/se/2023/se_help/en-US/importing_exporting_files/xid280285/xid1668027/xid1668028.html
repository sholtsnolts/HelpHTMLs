Creo Elements Direct data migration

The Creo Elements Direct data migration tool, CreoEDDataMigration.exe, translates Creo Elements Direct part, assembly, and drawing files (*.sdpc,*.sdp,*.sda, *.sdac, *.bdl and *.pkg) to Solid Edge. The migration tool supports Creo Elements Direct, versions 19.0 and newer.

Creo Elements Direct data migration supports both 2D and 3D data migration. During 2D migration, drawing views maintain associativity with the 3D model and dimensions. During 3D migration, 3D models maintain component colors and user-defined groups.

The migration tool does not support the migration of the material table and holes database.

CreoED2SE.ini file

The settings used when importing Creo Elements Direct files are defined in the CreoED2SE.ini file, located in the Solid Edge2023\Preferences\Translators folder. If you edit this file, use extreme caution in setting these parameters. Errors made to this file can adversely affect the quality of the translation.

Migrated data is imported in synchronous or ordered mode, based on the Solid Edge Options page→Helper tab settings.

You can use the Export Drafts as PDF or DXF parameter in the CreoED2SE.ini file to specify how Creo Elements Direct drawing documents (.bdl) are migrated to Solid Edge.

The values for this parameter are:

  • PDF – Exports the Creo Elements Direct draft file to .pdf format.

  • DXF – Exports the .dxf file from Creo Elements Direct and imports the .dxf file into Solid Edge Draft.

  • Both – Exports both a .dxf and .pdf file.

Hole feature recognition

You can use the Create Hole Features on Migration parameter in the CreoED2SE.ini file to control how Solid Edge hole features are created during migration.

The default value for the parameter is 1. When the parameter is set to 1, hole features, including thread attributes, are created during the migration. Hole features are only applied to synchronous bodies. No hole feature is created when the migration is targeted for the ordered environment.

Note:

External threads/threaded shafts are not supported.

When the parameter is set to 0, no hole or thread attributes are captured or applied during migration.

Drawing View migration

After migration, Creo Elements Direct drawing files (*.bdl) maintain associativity with the migrated Solid Edge model. For example, changes made to the model get updated in the drawing views for the model. During the migration process a link is established between the resulting Solid Edge part and assembly files and the migrated Solid Edge drawing file.

The following applies when migrating Creo Elements Direct files:

  • Drawing views are grouped with dimensions and labels.

  • All drawing views other than the dependent general views and partial views are migrated associative to models. The dependent general views and partial views is extracted geometry, non-associative to the model.

  • Cutting plane lines, auxiliary view plane lines, and detail envelopes are supported.

  • Dimensions migrated from Creo Elements Direct are attached to the geometry. Single, datum long, and chain dimensions are true dimensions after migration. Other dimensions are blocks.

  • Annotations, such as balloons and callouts, do not migrate as Solid Edge annotations, but migrate as blocks.

  • Center marks and center lines are connected to the Solid Edge drawing views.

  • Only automatically created hatches in Creo Elements Direct are migrated as associative hatch. Manual hatch migrate as blocks.

  • View shading is maintained as part of this drawing view creation process

    Note:

    Multi-sheet drawings are supported through this migration process.
