Migrate SolidWorks hole definitions to Solid Edge using the DOS window

When you migrate any SolidWorks part file the hole definitions including thread attributes are migrated.

Solid Edge provides following utilities to the Solid Edge program folder:

  • SEHolePipeTxtFromSW.exe—converts the SolidWorks hole database (swbrowser.sldedb to a Solid Edge Holes and pipes Excel spreadsheet (.xls).

  • SEHolePipeTxtFromSW_Legacy.exe—converts the SolidWorks hole database (SWBrowser.mdb to a Solid Edge Holes and pipes Excel spreadsheet (.xls).

To run the data migration tool

  1. Open a DOS window.

  2. CD to the Solid Edge Data Migration\Component\Program due to dependencies found only in this folder.

  3. Run SEHolePipeTxtFromSW.exe (to migrate sldedb database) or SEHolePipeTxtFromSW_Legacy.exe (to migrate mdb database) and set the following:

    • Argument 1: Holes or Pipes—Specifies if you are creating a hole file or pipe file.

    • Argument 2: [PathName1 + FileName]—The first path name is an sldedb file (swbrowser.sldedb) or mdb file (SWBrowser.mdb).

    • Argument 3: [PathName2]—The second path name is the target location where the Excel files will be placed. Do not include the name of the output file.

    • Argument 4—Units standards such as AI, DIN, and ISO within the swbrowser.sldedb or SWBrowser.mdb file to be converted.

      The following standards are available:

      Standard

      Input to the conversion utility

      ANSI Inch

      ai

      ANSI Metric

      am

      BSI

      bsi

      DIN

      din

      GB

      gb

      IS

      is

      ISO

      iso

      JIS

      jis

      KIS

      ks

      MIL

      mil
      Note:

      While migrating .mdb database, two files will be created for each database that is Inch and Metric.

      For .sldedb only one relevant database will be created that is either Inch or Metric file will be created based on the database.

After the Solid Edge hole database file (.xls) is created:

  1. Ensure that the Solid Edge Options\Files locations\Holes database is set properly

  2. Modify the SolidWorks2SE.ini to include the proper mapping to the Solid Edge hole database you just created.

    Example for a DIN Standard:

    [Hole Table mapping]

    swStandardDIN=SWBrowser_din_Metric (do not include file extension)

Note:

When using the utility, consider the following:

  • After successfully running the utility, ensure that the Location entry for the Holes and Pipes Excel spreadsheet (.xls) on the File Locations page of the Solid Edge Options dialog box is correct.

  • The creation of hole features and thread attributes assumes that the SolidWorks part files will be migrated to the Solid Edge synchronous environment.
