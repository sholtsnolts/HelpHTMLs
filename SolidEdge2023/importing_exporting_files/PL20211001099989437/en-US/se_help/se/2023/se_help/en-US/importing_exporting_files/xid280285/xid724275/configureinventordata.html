Inventor data migration

The Inventor Data Migration tool translates Inventor part (.ipt), assembly (.iam), and drawing (.dwg and .idw) files to Solid Edge.

To use the migration tool, Inventor and Solid Edge must be installed on the same computer. Solid Edge supports migration from Inventor version 2018 and above.

We recommend migrating the Inventor material and hole databases before you migrate Inventor CAD files.

The imported data is placed as either synchronous or ordered, depending on the settings in Open in on the Inventor Data Migration Start Page.

Hole feature recognition

To create hole feature during migration, select the Create hole features on Migration option on the Inventor Data Migration Start Page.

Note:

Definition for threaded shaft features is not supported

Holes other than non-threaded holes are not recognized in Solid Edge 2018. In Version 2019, all types of holes that are available in Inventor are recognized after Inventor data migration.

Hole features are only applied to synchronous bodies. No hole features are created when the data migration is targeted for the ordered environment.

Migrating drawing views

After migration, Inventor drawing files maintain associativity with the migrated Solid Edge model. For example, changes made to the model get updated in the drawing views for the model. During the migration process, a link is established between the resulting Solid Edge part and assembly files and the migrated Solid Edge drawing file.

To create drawing views during migration, select the Create drawing views on Migration option on the Inventor Data Migration Start Page.

The following applies when migrating Inventor files:

  • All Inventor drawing views other than - Overlay, Breakout, Slice and Crop are exported as associative views and appear as they do in Inventor.

    • For Overlay - Positions other than the main position are translated as dxf geometry. The main view is associative.

    • Breakout View - Any view containing a Breakout view is migrated as dxf geometry. Other Breakout views are not migrated.

    • Slice and Crop - These views are migrated as the parent view of Slice/Crop view in Inventor. The Slice/Crop action is eliminated from the view. Associativity is maintained between view and model

  • Cutting plane lines, auxiliary view plane lines, and detail envelopes are supported.

  • Migrated linear, radial, diametric, baseline and chain dimensions are associative. Other dimensions after migration are dxf geometry.

  • Centered Pattern - The circular center line that connects the various center marks is not migrated associatevely.

  • View shading is maintained as part of this drawing view creation process.

  • Annotations, such as balloons and callouts, do not migrate as Solid Edge annotations, but migrate as blocks.

  • Vertically or horizontally inherited break operations are migrated. Auxiliary or isometric views containing inherited break operations are not migrated.

  • Auxiliary or isometric views containing inherited broken out section operations are not migrated.

  • The angular dimensions are not migrated as true dimensions.

  • Dimension values are incorrect if they are attached to unsupported drawing views or entities. Dimension values that are attached to the dxf will be different in the migrated draft.

Migrating properties

Inventor files contain a set of properties that closely resemble Solid Edge file properties. During migration, these Inventor properties are mapped to Solid Edge file properties.
