Import Options for Inventor (.ipt and .iam) dialog box

This dialog box specifies how Solid Edge uses imported Inventor data. You can specify that it use the data as individual features or as a single body feature. You can also specify that Solid Edge heal or clean any inconsistencies in the surfaces or solids contained in the Inventor file you are importing. Inventor files must be saved in Inventor V6 or greater before they can be opened in Solid Edge.

Options

Specifies options to use during translation.

Generate Log File

Produces a log file that contains warnings and information about the files involved in translation. The name of the file is the <filename>.log and it is located in the same folder as the Inventor file.

Heal and Stitch

Specifies that you want to heal and stitch free surfaces to create a solid body. When given a set of sheet bodies the software cleans the input sheets of self-intersections and loop inconsistencies. It then identifies and removes very small surfaces or sliver sheets from the set of sheets. Edges of adjacent surfaces are then used to fill holes where slivers were removed. Larger holes from missing surfaces are also filled. After the healing process is complete a single stitch operation at 1.0e-5 (meters) is attempted to create a solid body.

Simplify Geometry

Specifies that whenever possible, you want replace the B-spline geometry from the model with simplified analytic geometry. This creates a smaller and more efficient model. This option also merges half cylinders that share a common center line.

If the system used to create the original model used a larger tolerance than the standard Parasolid tolerance (1.0e - 8), the translated model often contains a large number of tolerant edges and vertices. This simplification routine recalculates the model and repairs the edges and vertices to Parasolid precision.

Actions on Import

Specifies how you want to format the imported file.

Stitch Surfaces

Specifies that you want all surfaces and sheet bodies to be stitched to a tolerance of 1.0e-5 meters. It may be to your advantage not to stitch the surfaces on import, but to evaluate what needs to be stitched and perform the stitch after you import the file. This option is on by default.  If the stitching operation creates a valid volume, the volume is converted to a solid.

Boolean Solids

Specifies that you want to Boolean all solid bodies together to form a disjoint solid and added to PathFinder as a part copy. This option is on by default. If the option is off, all solid bodies are added as individual part copies to PathFinder.

Group Curves in a Single Part Copy

Specifies that you want to combine all curve data into a single part copy. It may be to your advantage to identify the curves that you do not need and either hide or delete them before importing the file. This option is on by default.

Body Check

Specifies that you want to perform a full body check on the file. This option is off by default.

Solids as design bodies

Specifies that you want to import the solid body as a design body.

When multiple bodies are translated into a single part file the process to make active or toggle from construction to design is the same for both Ordered and Synchronous.  To do this:

With Solids as design bodies enabled

  1. In PathFinder, select the body to which you want to add features.

  2. Click the right mouse button and select Activate Body.

With Solids as design bodies disabled

  1. In PathFinder, select the construction body that you wish to toggle to a design body.

  2. Click the right mouse button and select Toggle Design/Construction Body.

    Note:

    If the Make Base Feature parameter in the .ini file is enabled all solids are imported as design bodies and you can skip step 2. If the Make Base Feature parameter is disabled, you need to perform step 2 to toggle to the construction body to a design body.

  3. Click the right mouse button and select Activate Body.

If you clear this option, all solid bodies contained in the file you are importing are placed in PathFinder as Construction solids rather than a Design bodies.

Output Folder

Specifies an output folder for the documents you are importing. This is very helpful when you open foreign files with an assembly template. When Solid Edge encounters multiple bodies, the documents for each body are created and written to the specified folder with a .par extension. If you open single body foreign documents with a part template, Solid Edge does not automatically save the .par file to this folder. However if the log file is enabled it is written to this folder.

Output Folder is the Same as the Input Folder

Specifies that you want the output folder to be the same as the input folder for the documents you are importing. If you select this option and import an assembly, the assembly and the individual parts in the assembly are imported to the same folder.

Browse

Displays a dialog box that specifies an existing output folder or asks you to create a new output folder for the documents you are importing. This option is not available if the Output Folder is the Same as the Input Folder option is selected.
