Migrate the hole database from Creo Elements Direct to Solid Edge

When you migrate any Creo Elements Direct part file, the hole definitions for thread attributes are migrated. No database for non-threaded holes is migrated.

As part of the Creo Elements Direct data migration software, Solid Edge provides a tool that converts the Creo Elements Direct hole database, Werkst.csl, to a Solid Edge Holes and pipes Excel spreadsheet (.xls).

To run the data migration tool

  1. Open a DOS window.

  2. Enter the path for the SEHolePipeXMLFromCED.exe. For example, “C:\Program Files\Siemens\Solid Edge 2020\Program\SEHolePipeXMLFromCED.exe.”

  3. Set the following:

    • Argument 1: [PathName] — Location where migrated database must be saved. For example, “C:\Program Files\Siemens\Solid Edge 2020\Preferences\Holes.”

    • Argument 2: [Standard]—The unit standards to converted. Available standards include Metric, ANSI, British, DIN, Machining_Inch or Machining_Metric.

      After the process is complete, for all mentioned inputs, add the standards one at a time. All available default standards for threaded holes in Creo Elements Direct are migrated and are available in C:\Program Files\Siemens\Solid Edge 2020\Preferences\Holes.

      .

After the Solid Edge database is created:

  1. Ensure that the Solid Edge Options\Files locations\Holes database is set properly on the File Locations page of the Solid Edge Options dialog box.

  2. Modify the CreoED2SE.ini to include the proper mapping to the Solid Edge hole database you just created.

    Note:

    Migrate the hole database before attempting to migrate the Creo Elements Direct files.

Before migrating the Creo Elements Direct files, edit the following options in the [Hole Table mapping] section of the file $:\Program Files\Siemens\Solid Edge 2020\Preferences\Translators\CeoED2SE.ini.
