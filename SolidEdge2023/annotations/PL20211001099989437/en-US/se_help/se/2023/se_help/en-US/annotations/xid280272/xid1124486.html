Working with *legacy annotation saved settings

When you create new saved settings, they are saved to an external file, which you can share with other users. Before Solid Edge 2023, however, some saved settings only existed within, and were usable within, the file where they were created. When you open a file with these legacy saved settings, their names are marked by an asterisk (*)..

  • When you choose a command in a document where there are *legacy saved settings, you are prompted:

    Solid Edge has detected internally saved settings. These settings are indicated with an asterisk. It is recommended that you re-save these settings to your external file by clicking the save button on the Properties dialog box.

  • When you choose a legacy saved setting and attempt to make changes to it, you are prompted that it needs to be renamed to be saved. This removes the asterisk from the name and saves it to the external file. The legacy saved setting remains in the current document, unchanged.

    Example:

    In the Properties dialog box, when you choose an existing saved setting name and click the Save button, you are prompted:

    <saved setting name> already exists in the internal file. Rename the setting with a unique name before saving to the external file.

    Type a unique name in the Saved Settings box and then click Save.

  • After you rename and save your legacy settings to the external file, you can delete the legacy saved settings from your document. Select the name that is preceded by an asterisk, and then select the Delete button to remove it from the list of saved settings. To delete the information from the document, you must save the document.

    Note:

    You do not have to delete the legacy settings. You can retain them in the original document. For example, you may receive a document from someone who uses different annotation formatting than you do. You can reuse these saved settings within the document, but you cannot change them.
