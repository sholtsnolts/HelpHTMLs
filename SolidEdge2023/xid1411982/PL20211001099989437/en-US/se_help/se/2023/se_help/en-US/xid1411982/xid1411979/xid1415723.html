Solid Edge Requirements Management workflow

Use the options and commands located in the Requirements docking window to track requirements for mechanical systems. Use work items to describe criteria for your Solid Edge document. The following is an example of a workflow using Solid Edge Requirements Management.

  1. A product manager creates and assigns a requirement in Polarion for a mechanical designer to fulfill.

  2. Polarion emails the designer a notification containing the new work item information.

  3. In Solid Edge, the designer uses commands in the Requirements docking pane to search for the work item, and then add the assigned work item ID to the current document.

    The newly added work item ID is listed in the Requirements pane, along with other work item IDs already added to the document.

  4. When the work item is opened in Polarion, a comment indicates the Solid Edge document to which the work item ID has been added.

  5. The designer models the Solid Edge document to satisfy the criteria contained in the work item.

  6. In Solid Edge, the designer uses the Publish command in the Requirements docking pane to attach a 3D PDF of the document to the work item. When a draft document is published, a 2D PDF is attached.

    Polarion emails a notification to the product manager.

  7. The product manager opens the work item in Polarion, views the attached 3D PDF, and approves or rejects the design.
