Login tab, Manage page (Solid Edge Options dialog box)

The Login tab is a member of the tab set available in the Manage page of the Solid Edge Options dialog box. Use options on this page to affect Teamcenter login operations.

Teamcenter Environments
Select a Teamcenter environment

Specifies the default Teamcenter environment. For optimal performance, create the environment using Teamcenter Client Communication System (TCCS), then remove any existing URL using the Define Teamcenter Environments dialog box.

Get TCCS environments

Displays a list of available Teamcenter environments that use TCCS.

Tip:

If you added a new environment using TEM, and you do not see the environment listed, stop and start Teamcenter's file client cache (FCC), to display your entry in the list.

Create or update the list of Teamcenter environments

Displays the Define Teamcenter Environments dialog box so that you can add, modify, or remove Teamcenter environments.

Go offline

Allows you to work without interacting with Teamcenter. This option is disabled until you have successfully logged in to Teamcenter.

For details, see Working Offline in the managed environment.

Teamcenter Communication System
FMS_HOME

Displays the path for your FMS_HOME variable.

Use Teamcenter model data cache

Enables the client model cache where Teamcenter caches data model information locally, for example, item types, relation type, and dataset type. Using the client model cache can improve performance when Solid Edge requests and receives data model or schema information without going to the server.

Teamcenter Rich Client - Open in Solid Edge
Use same Teamcenter server as Rich Client

Enables Solid Edge to use the same Teamcenter server session in use by the rich client. As a result, opening a Solid Edge document from the rich client does not require an additional login.

Note:

This option is only supported with Teamcenter 11.5 and above. Teamcenter client communication system (TCCS) must be configured to use this option.

Solid Edge and Teamcenter Logging

Displays the location for the Teamcenter Server Syslog.

Performance Journal Logging
Enable Teamcenter server performance journal (.pjl)

Captures Teamcenter application performance and should not be used with other logging.

Include SQL Logging (=bJT)

Captures SQL logging data.

Incident Report Logging
Enable debug logging level

Enables Debug level logging such as TALLog, TAL_Debug and client-side COMLOG logs.

Note:

This option must be enabled before you sign in to Teamcenter.

Hosted Active Workspace client logging

When Debug logging is enabled, Active Workspace logging can be enabled by selecting one of seven levels of log details.
