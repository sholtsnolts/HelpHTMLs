Macros

You can use macros to:

  • Combine multiple commands into one.

  • Automate a series of tasks.

  • Speed up routine tasks.

Creating macros

You can create macros using any programming tool that supports OLE automation. The most popular of these tools is Microsoft Visual Basic. You create macros and their .exe files outside of Solid Edge.

Adding a macro to the Solid Edge interface

If you use several macros, you may want to add a tab to the command ribbon to store your macros in one place. You also can add macros to the Quick Access toolbar. To learn more about macros, see Add a macro to the command ribbon.

  • By Keyin macro provides a method to enter any standard command line command and argument as a menu entry.

    For example:

    macro.exe -i <value> - where value is a number or string of characters as input to the macro

    OR

    notepad.exe file.txt - selecting the icon directly opens the specified .txt file

Displaying image icons with macros

If you create a macro with an image icon embedded in it, Solid Edge uses that image. If the macro does not contain an embedded image, Solid Edge does the following:

  • If the macro is a .exe or .dll, Solid Edge looks in the same directory for an .ico, .png, or .bmp image file—in that order, and applies the first image found.

  • If no image file is found with the macro file, Solid Edge queries the operating system for an associated image and uses it. This is the same image you see when browsing to the folder using Windows Explorer, when the folder view type is set to show image icons. You can define a different icon by holding down the control key while clicking the Add button. Solid Edge displays a palette of images you can define as the icon.

  • If the macro is a website, Solid Edge checks to see if the website has a favicon available. If so, it downloads and uses the favicon image.

Running macros in Solid Edge

You can use the Run Macro command on the File menu to run any program that has an .exe extension.
