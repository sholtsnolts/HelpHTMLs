Managing settings and preferences

Solid Edge makes it easy for you to collect information associated with settings and preferences from one machine and deploy that information to other machines.

Solid Edge can capture into an encrypted file the following types of settings:

  • Registry settings—Stores many of the user preferences and customizations.

  • Files from the AppData folder—Stores customized themes.

  • Files from the Preferences folder—Often stored on a server, accessed by multiple users.

  • Template files—Often stored on a server, accessed by multiple users.

By default, the system stores the file, SolidEdgeSettingsAndPreferences.sepref, in the My Documents folder.

After you capture the settings and preferences information, you can deploy it to one or many machines, as needed.

Depending on the scope of your customization effort, you can use the following tools and methods:

  • To capture and deploy settings and preferences or to restore a system to out-of-the-box defaults, use the Settings and Preferences Wizard. The wizard is available from the Start menu→Solid Edge 2023.

  • To deploy settings and preferences on startup or to capture settings and preferences on exit, use the functionality on the User Profile page. The User Profile page is available from the File menu→Settings tab→Options command.

  • To programmatically capture and deploy settings and preferences or restore systems to out-of-the-box defaults, use command line arguments, as explained in Manage settings and preferences programmatically.
