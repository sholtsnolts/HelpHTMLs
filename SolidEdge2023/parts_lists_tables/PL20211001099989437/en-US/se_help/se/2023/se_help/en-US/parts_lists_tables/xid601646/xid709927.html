Referencing block properties in tables

Displaying block names and block view names in a table

Tables that are created from blocks using the Block Table command display these columns by default:

  • Item Number

  • Block Name

  • Block View Name

  • Quantity

The Item Number, Block Name, and Quantity columns are populated automatically when you create the table. To generate the Block View Name data, you must create the table using the option, Block view list.

Displaying block labels in a table

If the blocks also contain block labels with names and values, you can choose to show the label information in columns in the block table. You can do this by selecting a label name from the Properties list on the Columns tab in the Block Table Properties dialog box, and then adding it to the table using the Add Column button.

Example:

  • Label Name

  • Category

  • Inventory ID

  • Notes

To populate the block label columns with the block label values, you must create the table using the following options:

  • List by block occurrence

    • List block or block view on separate row

Displaying block properties in a table

Using the same technique as for block labels, you can choose any of the other named block properties to display as table column headings by selecting their names from the Properties list. The property values are extracted when the table is created as a Block list or Block view list.

For more information, see Create a table from blocks.
