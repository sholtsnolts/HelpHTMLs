Web Publisher Wizard (E-mail Details)

The e-mail options sends a published archive to one or more e-mail recipients from within IPA.

Wizard Options
Send page by email to:

Enter the recipient or recipients of the archived published pages. The addresses may be names from the address book of the default e-mail application or they may be e-mail addresses in the standard form: user@domain (e.g.customersupport@managmentsolutions.com). Multiple addresses must be separated by semi-colons (;) and not with blank spaces.

Subject

The subject may contain any text that should appear in the subject of the e-mail message.

Message:

The message text in this box will be the body of the e-mail message. The zipped archive file will be sent as an attachment. Any text explaining how to use the attachment should be entered in this box. The message may contain preset text if the Encrypt option (see below) is selected.

Include IPA WebView viewer

All web pages require the IPA WebView viewer (an ActiveX control) to be installed on the machine before viewing the page. For the convenience of the recipient of the published pages, this control can be bundled into the self-extracting archive and delivered in the e-mail message. It will be extracted and installed when the e-mail recipient opens the attachment.

Encrypt attachment to pass through firewalls

Many corporate firewalls strip executable files or archive files containing executable or JavaScript entries prior to delivering the message to the intended recipient. To avoid modification to the self-extracting archive, IPA will encrypt the file and change the extension to .ex_. This requires the user to rename the file back to .exe prior to running the self-extractor. When this option is selected, a note explaining this is added to the e-mail message and shows in the message box on this wizard page.
