Publishing internal components

When you create an internal component, components of that file are embedded in the Solid Edge assembly file being created, and no files are created on disk for the components within the assembly. If, for modifying or using in a downstream application such as draft, or if for any other reason you need the components on disk, use the Publish Internal Components command. The command maintains the document structure of the imported assembly file when publishing the internal components.

Selecting the Publish Internal Components command displays the Publish Internal Components dialog box. Use the dialog box to change the location where the file is to be saved, the template used to create the Solid Edge file, and to assign material to new part files that do not already have a material assigned.

You can make bulk changes to:

  • File location

  • Template

  • Material

After setting the appropriate options on the Publish Internal Components dialog box, click Publish to write the component files to disk.
