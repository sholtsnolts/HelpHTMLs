Item Number Status command

Queries for any parts lists that use assembly item numbers and compares the item numbers to Teamcenter Find Numbers to determine any out-of-date item numbers. The results are displayed in the Item Number Status dialog box (Draft) or Item Number Status dialog box (Assembly), and an associated report.

When you select the Item Number Status command, a report is generated showing the item number values in Draft or Assembly and corresponding values in Teamcenter. The report displays a parent and then its immediate children, and only shows the instances where Item Number is not equal to Find Number.

The report's default location, C:\Users\<username>\AppData\Roaming\Siemens\Solid Edge\Version 223\Log Files, is defined on the File Locations page of the Solid Edge Options dialog box. The report uses the naming convention ItemNumberStatusReport_<ItemID>-<Revision>_<timestamp>.csv where the Item ID and Revision are for the direct document. The timestamp is a 14 digit unique identifier using the format YYYYMMDDHHmmss where:

  • YYYY is the year. For example, 2015.

  • MM is the month. For example, 05 is May.

  • DD is the day. For example, 14 is the 14th.

  • HH is the hour using a 24-hour clock beginning at midnight which is 00 and going through 23, which is 11:00pm. For example, 13 is 1:00pm.

  • mm is the minute. For example, 12 is 12 minutes past the hour.

  • ss is the seconds. For example, 32 is 32 seconds past the minute.

Tip:

Report files persist, so you need to work with your system administrator to develop a schedule to archive the contents of the folder and free space on your computer.
